# 酒桌游戏小程序 - 社交功能设计文档

> 版本：v1.0
> 日期：2026-03-25
> 阶段：第三阶段（社交与多人互动）

---

## 一、功能概述

### 1.1 目标

增加小程序的用户粘性和传播能力，让用户可以：
- 发现更多有趣的卡组
- 与朋友远程一起玩
- 分享自己的创意卡组

### 1.2 核心功能

| 功能 | 说明 |
|-----|------|
| **卡组广场** | 浏览、发现、收藏其他用户的卡组 |
| **房间模式** | 创建/加入房间，实时同步抽卡状态 |
| **个人主页** | 展示我的收藏、我的发布、贡献值 |

## 二、卡组广场

### 2.1 页面结构

```
pages/square/square (卡组广场首页)
├── 顶部：搜索框 + 分类筛选
├── 横向：热门标签（搞笑、真心话、整蛊...）
├── 瀑布流：卡组卡片列表
└── 底部：发布卡组按钮

pages/square/detail (卡组详情页)
├── 卡组信息（名称、描述、创建者）
├── 卡牌预览列表
├── 收藏/使用按钮
└── 评论区域

pages/square/publish (发布卡组)
├── 基本信息（名称、描述、标签）
├── 图标选择
├── 添加卡牌
└── 发布/存为草稿
```

### 2.2 数据模型

```javascript
// 卡组
{
  id: string,           // 唯一ID
  name: string,         // 名称
  description: string,   // 描述
  icon: string,         // 图标 emoji
  tags: string[],       // 标签 ['搞笑', '真心话']
  cards: string[],       // 卡牌内容数组
  author: {
    id: string,
    name: string,
    avatar: string
  },
  likes: number,        // 收藏数
  uses: number,        // 使用次数
  createdAt: timestamp,
  isPublic: boolean    // 是否公开
}

// 评论
{
  id: string,
  deckId: string,
  content: string,
  author: { id, name, avatar },
  createdAt: timestamp
}
```

### 2.3 API 设计

| 接口 | 方法 | 说明 |
|-----|------|------|
| `/api/square/list` | GET | 获取卡组列表（支持分页、搜索、标签筛选） |
| `/api/square/detail/:id` | GET | 获取卡组详情 |
| `/api/square/publish` | POST | 发布卡组 |
| `/api/square/like` | POST | 收藏/取消收藏 |
| `/api/square/comments` | GET/POST | 评论列表/发表评论 |

### 2.4 交互流程

```
用户A 发布卡组
    ↓
后端存储 + 返回卡组ID
    ↓
用户B 访问广场
    ↓
浏览/搜索卡组
    ↓
查看详情 → 收藏/使用
    ↓
使用：该卡组导入到用户B的"我的卡组"
```

## 三、房间模式（多人同步）

### 3.1 页面结构

```
pages/room/home (房间首页)
├── 创建房间按钮
├── 加入房间输入框
├── 历史房间记录
└── 在线人数展示

pages/room/game (游戏房间)
├── 房间信息（房间号、在线人数）
├── 成员列表（头像 + 状态）
├── 公共卡池（当前使用的卡组）
├── 当前回合玩家高亮
├── 抽卡区域（只有当前玩家可操作）
├── 历史记录（谁抽到了什么）
└── 退出房间按钮
```

### 3.2 数据模型

```javascript
// 房间
{
  id: string,           // 房间号（6位数字）
  ownerId: string,      // 房主ID
  deckId: string,      // 使用的卡组ID
  status: 'waiting' | 'playing' | 'finished',
  members: [{
    id: string,
    name: string,
    avatar: string,
    isReady: boolean,
    isOwner: boolean,
    penaltyCount: number  // 被惩罚次数
  }],
  currentTurn: number,  // 当前成员索引
  gameLog: [{          // 游戏记录
    memberId: string,
    card: string,
    timestamp: number
  }],
  createdAt: timestamp
}
```

### 3.3 核心流程

```
1. 房主创建房间
   ├── 生成6位房间号
   └── 选择卡组

2. 玩家加入房间
   ├── 输入房间号
   └── 房主同意/自动加入

3. 游戏流程
   ├── 房主开始游戏
   ├── 系统按顺序通知当前玩家
   ├── 当前玩家点击抽卡
   ├── 其他成员看到抽卡结果
   └── 循环直到结束

4. 结算
   ├── 显示惩罚排行榜
   └── 可分享战绩
```

### 3.4 实时同步方案

**方案 A：WebSocket（推荐）**
- 实时性强
- 需要后端支持 WebSocket

**方案 B：微信实时音视频（TRTC）+ 状态同步**
- 可实现语音聊天
- 成本较高

**方案 C：HTTP 轮询（简易实现）**
- 每 2-3 秒拉取状态
- 延迟可接受，实现简单

| 行为 | 贡献值 |

## 五、技术实现要点

### 5.1 后端架构（建议）

```
┌─────────────┐     ┌─────────────┐
│   微信小程序  │────▶│   API 网关   │
└─────────────┘     └─────────────┘
                           │
        ┌──────────────────┼──────────────────┐
        ▼                  ▼                  ▼
┌─────────────┐     ┌─────────────┐     ┌─────────────┐
│  用户服务    │     │  广场服务    │     │  房间服务    │
│  (Auth)     │     │  (Square)   │     │  (Room)     │
└─────────────┘     └─────────────┘     └─────────────┘
        │                  │                  │
        ▼                  ▼                  ▼
┌─────────────┐     ┌─────────────┐     ┌─────────────┐
│  MySQL      │     │  MySQL      │     │  Redis      │
│  (用户数据)  │     │  (卡组数据)  │     │  (房间状态)  │
└─────────────┘     └─────────────┘     └─────────────┘
```

### 5.2 数据存储

| 数据 | 存储方式 | 说明 |
|-----|---------|------|
| 用户信息 | MySQL | 昵称、头像、贡献值 |
| 卡组数据 | MySQL | 广场卡组 |
| 房间状态 | Redis | 实时状态，断线自动清理 |
| 收藏关系 | MySQL | 用户ID + 卡组ID |

### 5.3 关键接口

```javascript
// 用户
POST /api/auth/login      // 微信登录
GET  /api/user/profile    // 获取个人信息
PUT  /api/user/profile    // 修改个人信息

// 广场
GET  /api/square/list    // 卡组列表（分页）
GET  /api/square/:id     // 卡组详情
POST /api/square         // 发布卡组
POST /api/square/like    // 收藏
GET  /api/square/my      // 我的发布/收藏

// 房间
POST /api/room/create   // 创建房间
POST /api/room/join     // 加入房间
GET  /api/room/:id      // 房间状态
POST /api/room/draw     // 抽卡
POST /api/room/next    // 下一回合
POST /api/room/leave    // 离开房间
```

## 八、总结

第三阶段的核心是**社交+多人互动**：

1. **卡组广场** — 让好内容被发现，让用户有成就感
2. **房间模式** — 远程聚会的核心场景
3. **个人主页** — 用户资产沉淀，贡献值激励

建议先做 **卡组广场**（纯内容展示，无实时要求），
再逐步实现 **房间模式**（需要实时同步，有技术难度）。
